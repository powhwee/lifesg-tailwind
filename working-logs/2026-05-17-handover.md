## Where each layer of the pyramid actually catches LifeSG-parity vs regression

This is the conceptual gap the late-session spot-check exposed. From `pilot-findings.md §2026-05-17`:

| Layer | Compares against | What failure means | Coverage today |
|---|---|---|---|
| L1 smoke | HTTP 200 + DOM mount + console | Render broke or threw | All 5 categories (~30 routes) |
| L2 measure | LifeSG's computed style + bbox | Real LifeSG-parity divergence | `/content/*` (6) + `core/typography` (1). **Forms, overlays, selection-and-input have NO L2 coverage.** |
| L3 behavioural | LifeSG's ARIA state under keyboard | Real behavioural divergence | Tab + Accordion only |
| L4 parity-spec visual snapshot | **Own previous render**, not LifeSG | Render has changed since baseline (regression) | 6 components — but the assertion is regression, not parity |
| L4 parity-spec axe | Axe rules at serious+critical | a11y violation | Same 6 components |
| L5 screenshot-all | Human eyeball | Whatever the reviewer flags | 57 routes — refreshes each run, no automated pass/fail beyond HTTP 200 |

The traps:
- L4 visual passing does NOT mean parity with LifeSG; it means no change since the baseline (which may itself be divergent).
- L2 passing in `/form/*`, `/overlays/*`, `/selection-and-input/*` is vacuous — there's no L2 there.

## Pitfalls to remember

(Carried forward from prior handovers — still relevant.)

1. **Tailwind 4 + Turbopack content-scan cache** — new `@theme inline` tokens may not generate utilities until a real edit to `globals.css`. See 2026-05-15.
2. **Root `.md` files are Tailwind sources by default.** If any new root doc contains `text-[var(--...)]`-shaped text, add it to `@source not` in `globals.css`.
3. **Next 16 blocks a second `next dev`** even on a different port. The Playwright config reuses port 3000.
4. **Probe before fixing the obvious.** See `pilot-findings.md §"Probe before trusting handover notes"`. Two punch-list items earlier in the week turned out to be wrong; the rule is to verify the target state itself, not just the diff classification.
5. **New:** Mistaking the parity-spec visual-snapshot test for a LifeSG-parity check. See the layer table above.

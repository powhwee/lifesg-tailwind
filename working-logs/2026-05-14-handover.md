## Rules to follow

These are non-negotiable, established in earlier 2026-05-13 entries and Gemini reviews:

1. **Classify every visible diff** as DEMO / COMPONENT / BOTH before opening a file. Source: `working-logs/pilot-findings.md` 2026-05-13 entry. ([details](pilot-findings.md))
2. **Verify before fixing.** A surprising number of "diffs" are screenshot artifacts (responsive layouts, label-wrap, viewport-width mismatches). Use `scripts/measure.mjs` or `snap-one.mjs` at 1500px before changing code.
3. **No new architectural debt.** Use clean utility syntax (`text-lifesg-text`), not arbitrary values (`text-[var(--lifesg-text)]`). When a token isn't in `@theme` yet, add it in the same commit. ([token-convention](pilot-findings.md))
4. **Boy-scout the file you touch.** Refactor any arbitrary-value tokens in that file as part of your change. Don't extend the debt elsewhere — that's separate work.
5. **Acceptable arbitrary values** (Gemini exception #1): runtime CSS variables set by libraries (`var(--collapsible-panel-height)`), micro pixel adjustments on UI lines (`outline-offset-[-2px]`), structural-container constraints (`max-w-[1200px]`). Annotate inline why.

## Pitfalls hit this session

1. **`Edit replace_all: true` doesn't always hit all instances.** If the new_string contains the old_string as a substring, only one occurrence may replace. Always `grep -c "old_pattern" file` after.
2. **Tailwind 4 `-outline-offset-2` is -8px, not -2px.** The default spacing scale is `--spacing: 0.25rem`, so `2` = 0.5rem = 8px. For -2px, keep `outline-offset-[-2px]` arbitrary with a comment — Gemini-exception #1 (micro UI line adjustment).
3. **`scripts/detect-tweaks.sh` and `working-logs/*.md` contain literal `text-[var(--...)]` placeholder text.** Tailwind 4 source-scans these by default and crashes on the `...`. `globals.css` already has `@source not` directives for both paths — keep them.
4. **`firebase apphosting:rollouts:create` requires a connected GitHub repo.** Without one, you get "Backend phapp is missing a connected repository." Use `firebase deploy --only apphosting` for local-source deploys.
5. **CSS `color-mix` percentages are non-linear visually.** I bumped `--toggle-bg-checked` from 5% to 12% to get a visible LifeSG-style tint. Below 10% reads as nearly white.
6. **Carousel screenshots from `snap-one.mjs` (1500px) and `screenshot-all.mjs` (1280px) have different widths.** Don't mix them — `screenshots/` is canonical at 1280px.

4. **Confirm the diff is real** (not a known false positive).
5. **Open the section file first** to classify DEMO/COMPONENT/BOTH (`src/components/{cat}/sections/{component}-default.tsx`).
6. **Fix the right layer** following the token convention. Add `@theme` entries before using utility classes.
7. **Boy-scout** any arbitrary-value tokens in the file you touched.
8. **Verify** with `npx tsc --noEmit` + a fresh snap.
9. **Commit** with a descriptive message — see prior commits for format.
10. After several commits, run `firebase deploy --only apphosting --project phapp-93b45 --force` to ship.

### Build trap discovered (worth flagging)

`README.md` line 215 contained a literal `text-[var(--...)]` string in a documentation table. Tailwind 4 auto-scans `.md` files at the project root for class candidates, generated CSS rule `.text-\[var\(--\.\.\.\)\] { color: var(--...); }`, and crashed the entire dev server with a CSS parse error.

**Fix**: added `@source not "../../README.md"` to `globals.css` (alongside the existing `scripts/` and `working-logs/` exclusions). Comment in `globals.css` also tightened so it no longer includes the literal placeholder inline.

If you add any other docs at the root (`AGENTS.md`, `CLAUDE.md`, etc.) that might contain `tw-utility-shaped` strings, add them to the exclusion list.

## Pitfalls to remember next session

1. **Tailwind 4 + Turbopack content-scan cache** — see 2026-05-15 handover. Still relevant when adding new `@theme inline` tokens.
2. **README + other root `.md` files are Tailwind sources** by default. If you write `text-[var(--...)]`-shaped placeholders in any new root doc, add it to the `@source not` list in `globals.css` or the dev server will refuse to compile.
3. **Next 16 blocks a second `next dev`** even on a different port. The Playwright config reuses port 3000; don't kill the dev server before running `npm test`.
4. **Probe before fixing the obvious.** Two punch-list items this session ("missing baseline hairline" on Navbar, "chevron in fixedCountry" on PhoneNumberInput) turned out to be wrong - LifeSG didn't actually have those behaviors. Always re-snap and probe with a small Playwright script before assuming the handover note is current.
